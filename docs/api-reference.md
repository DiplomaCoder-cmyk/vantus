# API Reference

This page groups the public Vantus API by responsibility so you can see where application composition ends and request processing begins.

## Composition Surface

| Type | Purpose | Notes |
| --- | --- | --- |
| `HostBuilder` | Main application composition root | Builds configuration, mounts modules, applies runtime limits, and produces an `ApplicationHost`. |
| `ApplicationHost` | Built application container | Can `run`, `run_blocking`, `serve`, or handle in-memory requests for tests. |
| `HostContext` | Lifecycle context for runtime modules | Exposes `AppConfig`, `RuntimeState`, `LogSink`, `IdGenerator`, and background task coordination. |
| `ServerHandle` | Running server control handle | Supports `shutdown()`, `local_addr()`, and `wait()`. |
| `HostBuildError` / `HostError` | Startup and runtime error types | Distinguish configuration, framework, module, and I/O failures. |

### `HostBuilder` methods you will use most

- `config_file(...)`
- `environment(...)`
- `profile(...)`
- `env_prefix(...)`
- `max_body_size(...)`
- `request_timeout(...)`
- `rate_limiter(...)`
- `log_sink(...)`
- `id_generator(...)`
- `module(...)`
- `group(...)`
- `compose_with_config(...)`
- `with_web_platform()`
- `with_observability()`
- `build()` / `try_build()`

## Module and Macro Surface

| Item | Purpose | Notes |
| --- | --- | --- |
| `#[module]` | Declares a route-bearing module impl | Generates `Module`; also generates `RuntimeModule` forwarding when lifecycle hooks exist. |
| `#[controller]` | Declares a route-bearing controller impl | Same route ergonomics without automatically opting into runtime-module semantics. |
| `#[get]`, `#[post]`, `#[put]`, `#[delete]`, `#[patch]`, `#[head]`, `#[options]` | Route annotations for impl methods | Must appear inside `#[module]` or `#[controller]` impl blocks. |
| `#[middleware(Type)]` | Attaches middleware to an impl block or route method | Middleware types must implement `Middleware` and `Default`. |
| `Module` | Base trait for routing + middleware | Generated by macros in normal application code. |
| `RuntimeModule` | Optional startup/shutdown lifecycle trait | Use for background tasks, readiness registration, or runtime initialization. |
| `Controller` | Marker trait for route-focused types | Blanket-implemented for all `Module` types. |

## Request and Response Types

| Type | Purpose |
| --- | --- |
| `RequestContext` | Per-request bridge to the request, path params, config, runtime state, request-local state, identity, logging, and ID generation |
| `Request` | Normalized framework request model |
| `Response` | Framework response model used by handlers and middleware |
| `Method` | HTTP method enum used by routing |
| `HttpError` / `FrameworkError` | Request-level and framework-level error types |

## Extraction and Request-Scoped State

### Core extractors

- `RequestContext`
- `Path<T>`
- `Query<T>`
- `Option<Query<T>>`
- `Header<T>`
- `Option<Header<T>>`
- `QueryMap`
- `BodyBytes`
- `TextBody`
- `JsonBody<T>`
- `RequestState<T>`
- `Option<RequestState<T>>`
- `IdentityState<T>`
- `Option<IdentityState<T>>`

### Supporting traits

| Trait | Purpose |
| --- | --- |
| `FromRequest` | Generic extraction contract for request-derived types |
| `NamedFromRequest` | Named extraction contract for path/query/header extractors |
| `OptionalFromRequest` | Optional extraction for request-scoped state and identity |
| `NamedOptionalFromRequest` | Optional named extraction for query/header |
| `IntoResponse` | Converts typed handler returns into `Response` |
| `IntoHandlerResult` | Compatibility shim over `IntoResponse` |

### Request-scoped extension points

- `RequestContext::insert_state(value)` and `RequestState<T>` for typed middleware-to-handler data flow
- `RequestContext::insert_identity(value)` and `IdentityState<T>` for caller identity propagation
- `Identity` as the marker trait for identity payloads

## Middleware Surface

| Item | Purpose | Notes |
| --- | --- | --- |
| `Middleware` | Core middleware trait | Receives the current `RequestContext`, delegates onward through the pipeline, and returns a handler-compatible future. |
| `MiddlewareStage` | Ordering bucket | `Logging`, `Recovery`, `Auth`, `Validation`, `Response`. |
| `MiddlewareFuture` | Shared future alias | Matches handler future shape. |

### Built-in middleware and platform modules

| Item | Purpose |
| --- | --- |
| `RequestLogger` | Logs request metadata through the configured `LogSink` |
| `SecurityHeaders` | Adds secure default headers to responses |
| `PanicRecovery` | Turns panics into `500 Internal Server Error` responses |
| `WebPlatformModule` | Installs `/health`, `/info`, and the default web hardening middleware |
| `ObservabilityModule` | Installs request IDs, metrics, readiness endpoints, and structured request logging |

## Public Design Boundaries

- Constructor injection is the intended dependency model for application services.
- Handler parameters are intentionally limited to request-derived values.
- TLS termination, CORS, compression, and auth helpers are not first-party runtime features in this release.
- Macro ergonomics sit on top of plain traits and builder APIs rather than replacing them.
